## PART 3: DEPLOY TO RENDER

### 3.1 Create Render Account
1. Go to https://render.com
2. Sign up with GitHub account (recommended for easy integration)
3. Verify your email

### 3.2 Create New Web Service
1. Click "New +" button â†’ "Web Service"
2. Connect your GitHub account if not already connected
3. Select your repository: `BreastCancer_Project_OgahVictor_22CG031902`
4. Click "Connect"

### 3.3 Configure Service Settings
```
Service Name: breast-cancer-predictor
Environment: Python 3
Build Command: pip install -r requirements.txt
Start Command: streamlit run app.py --server.port=10000
```

### 3.4 Configure Environment Settings
1. Click "Advanced"
2. Add Environment Variable:
   - Key: `STREAMLIT_SERVER_PORT`
   - Value: `10000`

3. Add another variable:
   - Key: `STREAMLIT_SERVER_ENABLECORS`
   - Value: `false`

4. Add another variable:
   - Key: `STREAMLIT_SERVER_HEADLESS`
   - Value: `true`

### 3.5 Deploy
1. Click "Create Web Service"
2. Wait for build to complete (3-5 minutes)
3. Once deployed, you'll see your live URL: `https://breast-cancer-predictor-xxxx.onrender.com`

### 3.6 Test Live Application
1. Visit the URL provided by Render
2. Test the prediction with sample data
3. Verify all features work correctly

## TROUBLESHOOTING

### Issue: Build fails on Render
**Solution:**
- Check requirements.txt syntax
- Ensure all package versions are compatible
- View build logs on Render dashboard

### Issue: App crashes after deployment
**Solution:**
- Check Render logs: Dashboard â†’ Select Service â†’ Logs
- Verify model files exist in correct location
- Test locally first before deploying

### Issue: Model files not found
**Solution:**
- Ensure model/ directory is in repository
- Files should be: `model/breast_cancer_model.pkl` and `model/scaler.pkl`
- Check file paths in app.py are correct

### Issue: Slow prediction
**Solution:**
- This is normal for Render free tier during first request
- Subsequent requests will be faster
